# 费率数据快速更新指南

## 🚀 超简单！只需3步

### 第1步：在飞书更新数据
1. 打开飞书文档
2. 修改费率数据
3. 确认"主力合约"标记正确
4. 保存

### 第2步：导出并上传Excel
1. 飞书文档 → 文件 → 导出 → Excel (.xlsx)
2. 保存到本地
3. 打开GitHub：https://github.com/skl-1/puble
4. 点击 "Add file" → "Upload files"
5. 上传新的Excel文件
6. 在 "Commit changes" 填写："更新费率数据"
7. 点击 "Commit changes"

### 第3步：等待自动更新
- ✅ GitHub Actions会自动检测文件变化
- ✅ 自动运行更新脚本
- ✅ 自动生成配置文件
- ✅ 自动提交到仓库
- ✅ **无需手动操作！**

**总耗时：3分钟**

## 📊 如何查看更新状态

### 方法1：查看GitHub Actions
1. 打开GitHub仓库
2. 点击顶部的 "Actions" 标签
3. 可以看到 "Update Fee Rates" 工作流
4. 查看运行状态：
   - ✅ 绿色 = 成功
   - ❌ 红色 = 失败

### 方法2：查看配置文件
1. 打开 `src/data/feeRates.ts`
2. 查看更新时间（文件顶部注释）
3. 查看品种数量是否正确

### 方法3：查看提交记录
1. 点击 "Commits"
2. 可以看到自动提交的记录
3. 格式：`chore: auto update fee rates from Excel [skip ci]`

## ⚠️ 常见问题

### Q1：上传后没有自动更新？
**A：** 检查以下内容：
1. 文件名必须是 `交易所标准.xlsx`
2. 必须上传到仓库根目录
3. 查看 GitHub Actions 是否在运行

### Q2：自动更新失败了？
**A：**
1. 打开 Actions 查看错误信息
2. 可能原因：
   - Excel格式不正确
   - "主力合约"标记缺失
   - Python依赖安装失败

**解决方法：**
- 在本地运行 `pnpm update-fees` 手动更新
- 查看详细错误日志

### Q3：可以手动触发更新吗？
**A：** 可以！
1. 打开 GitHub 仓库
2. 点击 "Actions" 标签
3. 选择 "Update Fee Rates" 工作流
4. 点击 "Run workflow"
5. 点击 "Run workflow" 按钮
